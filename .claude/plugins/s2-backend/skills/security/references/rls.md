# Row Level Security (RLS)

RLS is enforced at the database level via PostgreSQL session variables. `TenantAwareDataSource` injects the current `TenantContext` into every connection before the query executes.

## TenantAwareDataSource

Wraps the real `DataSource`. On every `getConnection()` call it sets two PostgreSQL session variables using `set_config()`.

```java
public class TenantAwareDataSource extends DelegatingDataSource {

    @Override
    public Connection getConnection() throws SQLException {
        Connection connection = super.getConnection();
        setTenantContext(connection);
        return connection;
    }

    private void setTenantContext(Connection connection) throws SQLException {
        TenantContext context = TenantContextHolder.getContext();

        boolean bypass = false;
        String tenantIdsStr = "";

        if (context != null) {
            bypass = context.rlsBypass();
            tenantIdsStr = String.join(",", context.tenantIds());
        }
        // null context → secure defaults: bypass=false, empty tenantIds (no data visible)

        try (PreparedStatement stmt = connection.prepareStatement("SELECT set_config(?, ?, false)")) {
            stmt.setString(1, bypassVarName);        // e.g. "app.rls_bypass"
            stmt.setString(2, String.valueOf(bypass));
            stmt.execute();

            stmt.setString(1, tenantIdsVarName);     // e.g. "app.unit_ids"
            stmt.setString(2, tenantIdsStr);
            stmt.execute();
        }
    }
}
```

**Session variable names** are configurable:
```yaml
s2grupo.tenant.sessionVars:
  bypass: "app.rls_bypass"
  tenantIds: "app.unit_ids"
```

## PostgreSQL side

```sql
-- Helper function (generated by RlsTestSupport.createTenantIdsFunctionSql())
CREATE OR REPLACE FUNCTION current_tenant_ids() RETURNS UUID[] AS $$
BEGIN
    IF current_setting('app.rls_bypass', true) = 'true' THEN
        RETURN (SELECT coalesce(array_agg(id), ARRAY[]::UUID[]) FROM unit);
    END IF;
    RETURN string_to_array(current_setting('app.unit_ids', true), ',')::UUID[];
EXCEPTION
    WHEN OTHERS THEN RETURN ARRAY[]::UUID[];
END;
$$ LANGUAGE plpgsql STABLE LEAKPROOF;

-- RLS policy on a table
ALTER TABLE my_table ENABLE ROW LEVEL SECURITY;
ALTER TABLE my_table FORCE ROW LEVEL SECURITY;
CREATE POLICY my_table_tenant_rls ON my_table
    USING (tenant_id_column = ANY(current_tenant_ids()));
```

## @WithRlsBypass — non-HTTP contexts

Methods that run outside an HTTP request (scheduled jobs, listeners, `@PostConstruct`) have no `TenantContext`. Annotate with `@WithRlsBypass` to temporarily set bypass mode via AOP.

```java
@Scheduled(cron = "${...}")
@WithRlsBypass
void resetApiKeys() { /* full visibility */ }

@RabbitListener(queues = "...")
@WithRlsBypass
void onMessage(MyEvent event) { /* full visibility */ }
```

The `RlsBypassAspect` saves the previous context, sets bypass, and restores on exit.

## TenantContextRunner — programmatic bypass

For code that cannot use annotations, inject `TenantContextRunner`:

```java
// During startup / CORS config / any Spring bean
String result = tenantContextRunner.supplyWithBypass(() -> service.getData());
tenantContextRunner.runWithBypass(() -> service.doWork());
```

## When to use @WithRlsBypass vs TenantContextRunner

| Context | Solution |
|---|---|
| `@Scheduled` job | `@WithRlsBypass` |
| `@RabbitListener` / `@KafkaListener` | `@WithRlsBypass` |
| `@PostConstruct` / `@EventListener(ApplicationReadyEvent)` | `@WithRlsBypass` |
| Startup code in a Spring `@Bean` | `TenantContextRunner.runWithBypass()` |
| Filter running before `TenantContextFilter` | Manual set + clear in `finally` |
| Normal HTTP controller request | Nothing — `TenantContextFilter` handles it |

## RlsTestSupport — integration testing

`RlsTestSupport` generates the SQL needed to set up and test RLS in integration tests:

```java
@BeforeAll
static void setup() throws SQLException {
    try (Connection conn = adminConnection()) {
        Statement stmt = conn.createStatement();
        stmt.execute(RlsTestSupport.createTenantIdsFunctionSql());
        stmt.execute(RlsTestSupport.enableRlsSql("orders", "unit_id"));
    }
}

@Test
void shouldFilterByTenant() throws SQLException {
    try (Connection conn = appUserConnection()) {
        RlsTestSupport.setTenantContext(conn, List.of("tenant-1"), false);
        // queries on conn now only return tenant-1 data
    }
}
```
